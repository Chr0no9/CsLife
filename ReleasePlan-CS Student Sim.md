## **High-Level Goals**

1. Develop a fully playable CS college life simulator in which players manage a student's college experience.  
2. Implement a character quarterly progression system with basic statics and player decision options  
3. Create the ability to allow for one complete playthrough from the beginning of freshman year to graduation with multiple possible endings

## **User-Stories**

### **Sprint 1- Core Game**

| Priority | User Story | Story Points |
| :---- | :---- | :---- |
| High | As a player, I want to start a new game in which I can choose my name and starting stats. | 5 |
| High | As a player, I want the ability to view my character’s stats. | 8 |
| Medium | As a player, I want to be able to progress through quarters. | 5 |
| Low | As a user, I want my progress to be saved. | 3 |

### **Sprint 2- Decisions & Events**

| Priority | User Story | Story Points |
| :---- | :---- | :---- |
| High | As a player, I want the ability to choose clearly to progress through the game all the way until graduation. | 8 |
| High | As a player, I want the ability to choose how I dedicate my time each quarter and those choices affect my statistics  | 5 |
| High  |  As a player, I want my game progress to be saved and loaded so that I can continue playing without  losing my character, statistics and quarter progression | 5 |
| High  | As a player, I want to play random mini-games in between the quarter | 8 |
| High | As a player, I want random life events to occur in between the quarters  | 8 |
| Low | As a player, I want an end game that has different graduation outcomes based on the choices and statistics gathered throughout college.  | 3 |
| Medium | As a team, we want a Github Scrum board to organize sprint work, assign tasks, and track the progress of each task throughout the sprint. | 1 |

### **Sprint 3- Mini Games & Career Progression**

### **Sprint 4- End Game**

| Priority | User Story | Story Points |
| :---- | :---- | :---- |

## **Sanity Check**

	The release plan is designed to deliver a viable product within the five week timeframe. The team will consist of 4-6 members in which work will be distributed across gameplay, UI development, game logic, and testing. Additional features will be added in the product backlog to avoid too big of a project scope.
